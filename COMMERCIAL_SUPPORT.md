# Commercial Support

Commercial support is available for organizations that want to integrate signed execution receipts and independent validation into agent workflows.

## Available support areas

1. Private validator integration
2. Custom receipt profile design
3. CI/CD receipt verification using the GitHub Action
4. CrewAI and agent workflow integration
5. FDO/Data-Space-like evidence mapping
6. Enterprise key-management adaptation planning
7. Internal demo preparation for audit, governance, or research review
8. Research collaboration and artifact preparation

## Typical integration work

Examples include:

- adapting the receipt schema to internal tool calls
- adding organization-specific validator checks
- integrating validator output into CI/CD or audit pipelines
- mapping internal resource identifiers to FDO/Data-Space-like identifiers
- preparing reproducible demos for technical review
- designing a migration path from demo signing to production key management

## Boundaries

Commercial support does not change the open-source project's security guarantees.

This project provides verifiable execution evidence for tool invocations. It does not prove semantic correctness of tool outputs. It does not prove that the policy itself is correct. It does not protect against a compromised signer. It does not replace sandboxing, IAM, access control, monitoring, human approval, or legal compliance review.

This project is not a full FDO, Gaia-X, IDS, or EDC implementation.
